# 中文技能与接触受击反馈设计

## 1. 目标

解决当前试玩截图中最直接的两个可玩性问题：升级界面混用英文，且小怪贴近玩家时缺少可确认的伤害与受击反馈。

本阶段完成后，玩家应当能清楚读懂每个技能选项，并通过敌人前摇、玩家生命变化、角色闪烁、冲击火花和伤害数字确认一次接触攻击。敌人被玩家武器击中时也必须有独立反馈，但不能遮盖敌人的高威胁攻击预警。

## 2. 当前证据与问题边界

- `core_upgrades.json`、五把现有武器、升级面板标题和效果摘要均使用英文显示文本。
- 小怪数据已配置 `contact_damage: 8`，玩家已有 `0.6` 秒接触伤害无敌时间。
- 当前测试直接调用接触伤害函数，没有实例化真实玩家与敌人场景，也没有覆盖碰撞边缘位置。
- 敌人开始攻击和确认命中都使用“双方碰撞半径之和”作为精确距离。角色物理碰撞可能将双方分隔在该边界之外，造成视觉上已经贴身但攻击状态无法稳定命中。
- 玩家和敌人目前都没有独立受击反馈组件。仓库中的静态正面受击图不能直接用于侧面或背面移动状态，否则会破坏朝向和动作一致性。

## 3. 已确认方案

采用“保持朝向的分层受击反馈”。不切换到静态正面受击姿势，而是在当前动画帧上叠加短时反馈：

1. 小怪按统一动作契约执行 `移动 -> 前摇 -> 有效帧 -> 恢复`。
2. 有效帧成功命中后，玩家扣除伤害并进入接触无敌时间。
3. 玩家当前动画保持原朝向，同时显示红色闪烁、红橙冲击火花、生命文本脉冲和向上飘出的伤害数字。
4. 敌人受到玩家武器伤害时保持当前动作，同时显示亮色闪烁和金青色命中火花。
5. 敌人攻击预警继续使用 `modulate`；受击闪烁只使用独立覆盖层或 `self_modulate`，不得抹掉前摇、有效帧和恢复帧颜色。

## 4. 中文显示范围

### 4.1 必须中文化

- 升级面板标题：`选择强化`。
- 升级面板副标题：`战斗暂停`。
- 当前五把武器的名称和说明。
- `core_upgrades.json` 中全部可选升级名称。
- 升级效果摘要，例如 `伤害 +15%`、`冷却 -6%，移速 +12`、`武器等级 +1`、`解锁武器`。
- 选择后的 HUD 提示，例如 `已选择：疾行敕令`。
- 升级按钮缺项占位显示为 `无可用强化`，不得显示英文 `Upgrade`。

### 4.2 保持稳定的内部字段

- 内容 ID、JSON 键、脚本类名和信号名继续使用英文 ASCII。
- 数值和百分号保持阿拉伯数字，便于移动端快速扫描。
- 本阶段不建立完整多语言切换框架；显示数据直接改为简体中文。未来国际化以稳定 ID 为键迁移，不修改本阶段的内部接口。

### 4.3 字体策略

本阶段使用 Godot 的系统字体回退显示中文，并在 Windows 可视化烟测中检查所有字形。Android 发布质量阶段再决定是否打包固定 CJK 字体，以避免本阶段为单一界面引入大体积字体资源。

## 5. 接触伤害规则

### 5.1 攻击触达距离

- 新增统一的接触攻击触达余量，默认 `4.0` 逻辑像素。
- 攻击开始距离和有效帧命中距离必须调用同一计算函数：`敌人半径 + 玩家半径 + 触达余量`。
- 触达余量只补偿物理接触边缘，不扩大为远距离攻击。
- `basic_demon` 的单次伤害保持 `8`，不在本阶段调整敌人伤害数值。

### 5.2 动作与伤害约束

- 前摇阶段不造成伤害。
- 每次有效帧最多命中一次。
- 恢复阶段不造成伤害。
- 多个敌人同时攻击时仍受玩家全局 `0.6` 秒接触无敌保护，避免瞬间叠加致死。
- 玩家死亡后拒绝后续接触伤害并停止受击反馈。
- 敌人死亡或回池后立即清除攻击状态和反馈状态。

## 6. 受击反馈组件

新增可复用的 `HitFeedback` 组件，挂载在玩家与通用敌人场景下。组件只负责表现，不计算伤害。

### 6.1 组件接口

- `configure(target: CanvasItem, spark: Sprite2D, damage_label: Label = null)`：绑定表现节点。
- `play_hit(amount: int, profile: StringName)`：播放 `player` 或 `enemy` 反馈。
- `reset_feedback()`：回池、死亡或重新开始时恢复颜色、缩放、透明度和标签。
- `is_playing() -> bool`：供测试与场景状态检查使用。

`PlayerController` 与 `EnemyAgent` 各自在 `_ready()` 中只连接一次自身 `HealthComponent.damaged` 信号，并由该连接调用对应的 `HitFeedback`。`GameLoop._on_player_damaged()` 继续负责生命值同步，并调用 `HUD.show_damage_feedback(amount)` 播放 HUD 脉冲。

### 6.2 玩家反馈

- 角色在 `0.12` 秒内闪红再恢复，不隐藏当前动画。
- 冲击火花显示 `0.18` 秒，最大尺寸不超过玩家身体宽度的 `70%`。
- 伤害数字使用 `-8` 格式，在 `0.45` 秒内向上移动约 `24` 像素并淡出。
- HUD 生命文本在一次有效伤害后闪红并轻微放大，`0.18` 秒内恢复。
- 普通小怪命中不触发全屏停顿和相机震动。

### 6.3 敌人反馈

- 敌人覆盖层在 `0.08` 秒内亮白偏青并恢复。
- 命中火花显示 `0.12` 秒，尺寸不超过敌人身体宽度的 `55%`。
- 反馈不得修改敌人动作状态，不得延长或跳过前摇、有效帧、恢复和死亡。
- 同一次投射物不能对同一目标重复触发伤害或反馈。

### 6.4 对象复用

- 火花、覆盖层和玩家伤害标签均作为场景内常驻子节点复用，不在每次命中时创建节点。
- 敌人回池时调用 `reset_feedback()`，防止旧闪烁、缩放和标签状态泄漏到下一次激活。
- 高频命中会重新开始当前反馈，不排队积压动画。

## 7. 正式美术素材

使用内置 image-2 生成两张风格一致、便于抠图的纯绿幕源图：

1. `player_contact_hit_burst_green.png`：红橙色符纹冲击火花，用于玩家被妖物命中。
2. `enemy_weapon_hit_spark_green.png`：金色与青白色剑气火花，用于敌人被玩家武器命中。

源图进入 `art/source_green/effects/hit/`，抠图后的透明 PNG 进入 `art/effects/hit/`。素材必须无文字、无投影、无烟雾背景，主体居中并留足边距。接入前检查透明角、绿边、手机实际缩放和深色地面上的轮廓。

## 8. 数据与信号流

### 8.1 小怪命中玩家

1. `EnemyActionState` 从前摇进入有效帧。
2. `EnemyAgent` 使用统一触达距离确认玩家仍在范围内。
3. `PlayerController.take_contact_damage(8)` 返回 `true`。
4. `HealthComponent` 发出 `damaged(8)`。
5. `PlayerController` 的固定信号连接调用玩家 `HitFeedback.play_hit(8, &"player")`。
6. `GameLoop` 更新 HUD 生命值，并调用 `HUD.show_damage_feedback(8)` 播放生命文本脉冲。

### 8.2 玩家命中小怪

1. 投射物确认目标尚未被本次攻击命中。
2. 目标 `HealthComponent.take_damage(amount)` 发出 `damaged(amount)`。
3. `EnemyAgent` 调用敌人 `HitFeedback.play_hit(amount, &"enemy")`。
4. 若生命归零，死亡状态优先，碰撞立即关闭并重置反馈。

## 9. 测试与验收

### 9.1 自动测试

- 使用真实 `Player.tscn` 与 `BasicDemon.tscn` 建立接触战斗测试。
- 双方中心距离为碰撞半径之和再加 `2` 像素时，小怪能够进入前摇，并在有效帧造成恰好 `8` 点伤害。
- 前摇和恢复阶段不扣血；同一有效帧不重复扣血；玩家无敌时间内拒绝第二次伤害。
- 玩家和敌人 `HitFeedback` 能开始、按时结束并恢复初始颜色、缩放和标签。
- 敌人回池再激活后没有残留反馈。
- 所有升级选择名称和效果摘要均为中文，且同一按钮不混入英文效果关键词。
- 升级面板标题、副标题、缺项占位和选择后提示为中文。

### 9.2 可视化验收

- 在 `720 x 1280` 竖屏下触发升级界面，三张卡片文字完整、不溢出。
- 让至少五只小怪围住玩家，确认生命值下降、反馈清晰且不形成持续闪屏。
- 观察小怪前摇、命中和恢复颜色，确认玩家受击火花没有覆盖敌人预警。
- 用飞剑连续命中敌人，确认敌人受击闪烁不会中断移动或攻击动作。
- 在深色荒土地面上检查两张火花素材的轮廓与绿边。

### 9.3 回归门槛

- Godot 全量测试通过。
- 主场景连续运行 `600` 帧无脚本错误、物理错误或资源错误。
- 中文升级界面和接触受击场景各保留一张可视化截图作为本阶段证据。

## 10. 非目标

- 不在本阶段实现模块化武器分支、状态反应或武器进化。
- 不新增完整受击姿势动画，不使用静态正面图覆盖侧面或背面动作。
- 不调整武器伤害、升级权重、刷怪速度或 Boss 数值。
- 不加入相机震动、全屏停顿、复杂伤害数字系统或完整多语言设置。

## 11. 完成标准

- 试玩截图中出现的升级选项全部使用简体中文名称和中文效果摘要。
- 小怪在真实场景碰撞边缘可以稳定执行前摇并造成 `8` 点接触伤害。
- 玩家和敌人都具有方向一致、可读且不会遮盖攻击预警的受击反馈。
- 两张 image-2 正式火花素材完成抠图、接入和实际缩放检查。
- 自动测试、可视化检查和 `600` 帧场景烟测全部通过。
