# 角冲妖冲锋预警设计

## 目标

让第一章角冲妖形成清晰的横向闪避问题：玩家能在手机屏幕上看懂冲刺方向、实际威胁长度和伤害窗口，但不能站着等怪慢慢靠近。

## 数值

- 触发距离：300 像素。
- 前摇：0.55 秒，期间停步并锁定起手时的方向。
- 冲刺速度：420 像素/秒。
- 有效时间：0.70 秒，对应实际冲刺长度 294 像素。
- 恢复：0.75 秒；伤害 14。
- 玩家在前摇期间约可移动 143 像素，足够横向躲开但不能无视。

## 视觉

- 前摇显示双层直线车道：宽而半透明的红色危险带，加一条亮橙中心线。
- 车道长度严格等于 `charge_speed * attack_active`，端点放置角冲符印。
- 有效帧隐藏预警，显示与方向一致的冲刺残影。
- 恢复阶段短暂显示尘爆，角色本体沿用现有灰色恢复提示。
- 正式符印、残影和尘爆由内置 image-2 生成绿幕源图，使用红橙威胁色。

## 技术边界

- 方向在前摇开始时锁定，玩家之后移动不会让冲刺无提示转向。
- 视觉节点复用 `BasicDemon` 场景，只有 `behavior == "charge"` 时显示。
- 冻结暂停前摇和有效时间；封印禁止开始新的冲锋。
- 死亡、对象池释放、重新配置为其他敌人时必须隐藏所有冲锋视觉。

## 验收

- 前摇车道长度与实际冲刺距离一致。
- 大步长跨越动作状态时不重复冲刺或遗留视觉。
- 方向锁定、冻结、封印和池复位测试通过。
- 720x1280 竖屏中预警不被背景吞没，也不与玩家青蓝武器混淆。
