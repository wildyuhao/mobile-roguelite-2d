# 移动端战斗 HUD 可读性设计

## 目标

让玩家在竖屏战斗中不用停下来读小字，就能快速判断剩余生命、升级进度、当前等级和战斗时间。HUD 保持紧凑，不遮挡中央战斗区域，也不与右上角暂停按钮冲突。

## 信息层级

- 第一行显示战斗时间和当前等级：时间字号 `24`，等级字号 `18`。
- 第二行是生命进度条，固定高度 `32`，文字显示“生命 当前 / 最大”。
- 第三行是灵气进度条，固定高度 `26`，文字显示“灵气 当前 / 所需”。
- 升级选择反馈位于进度条下方，使用绿色短提示，不改变状态带尺寸。

## 布局

- 状态带锚定左上角，左侧和顶部各留 `24` 像素安全边距。
- 最大宽度 `380` 像素，不进入右上角暂停按钮的触摸区域。
- 使用稳定的最小尺寸与间距；数值变化不得推动战斗画面或改变暂停按钮位置。
- 在 `720 x 1280` 与 `390 x 844` 竖屏上均保持完整可见。

## 视觉

- 生命条使用深红填充与暗色半透明底，低血量依然通过长度和数字双重表达。
- 灵气条使用青蓝填充，与经验拾取物和武器冷色反馈保持一致。
- 文字使用中文“等级 / 生命 / 灵气”，时间保留 `MM:SS` 数字格式。
- 使用 Godot `ProgressBar`、`Label` 和 `StyleBoxFlat` 原生控件；这是结构化 UI，不额外生成位图素材。

## 行为

- `set_health` 同时更新生命条的 `max_value`、`value` 和数字标签。
- `set_experience` 同时更新灵气条的 `max_value`、`value` 和数字标签。
- 受伤时生命数字和生命条共同产生短促放大、红色强化反馈，结束后恢复。
- 升级和手动暂停期间 HUD 数值保持可见，但不抢占面板交互。

## 验收

- 生命和灵气条存在，尺寸满足移动端可读性要求。
- 中文数值、进度值和最大值始终同步。
- 受伤反馈结束后恢复基础缩放和颜色。
- HUD 不覆盖右上角暂停按钮，长屏下仍锚定左上。
- Godot 全量测试、600 帧烟测和标准/长屏可视化检查通过。
