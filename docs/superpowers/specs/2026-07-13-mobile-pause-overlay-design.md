# 移动端手动暂停设计

## 目标

为竖屏战斗补齐随时可触达的手动暂停入口。玩家可以在不中断本局构筑的前提下暂停、继续或重新挑战；升级选择和战斗结算仍各自拥有独立的暂停状态，不与手动暂停叠加。

## 交互

- 常规战斗时，右上角显示 `56 x 56` 的暂停图标按钮，距顶部和右侧各 `24` 像素。
- 点击后立即暂停场景树、清零虚拟摇杆移动输入，并显示全屏半透明遮罩。
- 暂停层中央显示“战斗暂停”，提供“继续战斗”和“重新挑战”两个至少 `64` 像素高的触摸按钮。
- `Esc / ui_cancel` 可在常规战斗和手动暂停之间切换。
- 升级选择、胜负结算期间隐藏暂停入口，避免同时存在两个暂停层。

## 状态边界

- `manual_pause_active` 是手动暂停唯一状态标记。
- 只有游戏未结束且场景树未被其他流程暂停时，才能进入手动暂停。
- 恢复仅解除由手动暂停拥有的暂停状态。
- 重新挑战先解除暂停，再重新加载当前场景。
- 升级开始时暂停入口不可用；成功或失败退出升级选择后恢复入口。
- 玩家失败或首领结算时永久隐藏本局暂停入口。

## 视觉

- 图标使用内置 image-2 生成的正式位图素材：暂停、继续、重开三个熟悉符号。
- 图标采用深色石质轮廓、青蓝高光和少量金色边缘，与现有修仙废土界面协调。
- 所有图标从纯绿幕源图抠出，透明边角、无绿边、无文字和水印。
- 暂停层保持克制：深色遮罩、紧凑面板，不遮挡成卡片套卡片的结构。

## 验收

- 暂停层在暂停场景树后仍可响应按钮与键盘输入。
- 暂停、继续和重开信号各只触发一次。
- 主场景包含暂停层，三个图标纹理均已绑定。
- 手动暂停不会在升级或结算期间出现，也不会遗留摇杆移动。
- Godot 全量测试、对象池/遭遇模拟、600 帧烟测和 `720 x 1280` 可视化检查通过。
